Orkin.com or Orkincanada.ca is Down

The Marketing Technology team uses the website monitoring service, Pingdom, to monitor Orkin websites and issue alerts if a site is unresponsive.

If you suspect that an Orkin website is down, escalate this to the Marketing Technology team by emailing MarketingTechAlerts@Rollins.com.

The Marketing Technology team relies on experience and expertise to troubleshoot these issues, and may use tests and solutions such as:

  • Attempts to access the site from various devices and networks, to ensure the problem is not isolated to the Rollins network
  • Assessing specific errors, such as 404 Page Not Found and 403 Forbidden, for indications of the nature of the issue
  • Monitoring the AWS Dashboard status check logs for website and database instances
  • Reviewing the AWS Dashboard CloudWatch metrics for abnormal CPU activity
  • Rebooting the production website instances
  • Examining the public DNS, using the PuTTY console, to locate errors
  • Examining each application connected to the production website, to isolate the issue

Opening a Support Case for Amazon Web Services (AWS)

To open a support case in AWS, follow these steps:

  1. Navigate to https://rollins.signin.aws.amazon.com/console and log into the Rollins AWS account using the Marketing Technology credentials.
  2. Navigate to the Support Center.
  3. Click Create Case.
  4. Populate all relevant fields and provide as much detail as possible about the issue. Select the appropriate Service and Category.
  5. Select a contact method. If the severity of the problem is high, it is preferrable to select Phone as your contact method.
    IMPORTANT: Select the appropriate severity level. The severity level is critical to the turnaround time for AWS Support.
  6. Click Submit.

International Franchise Site is Inaccessible

If a franchisee, consumer, or Rollins employee is unable to access a franchise website, follow these troubleshooting steps:

  • Ensure that other websites can be accessed; for example, google.com and bbcnews.com.
  • Refresh the browser window. To force a full refresh, press Ctrl-F5.
  • Attempt to access the site using different browsers.
  • Clear the browser cache.
  • Attempt to access the site from a different network.
    • If the computer is using a Wi-Fi connection, connect to a different Wi-Fi network.
    • Use a mobile device to access the site.
    • Solicit feedback from coworkers in remote workspaces.


If these steps fail to correct the problem, contact the Marketing Technology team. Before investigating at the server level, the Marketing Technology team may attempt further troubleshooting prior to investigating the server, including:

  • Ping the website.
  • Use a web service such as www.isitdownrightnow.com. Navigate to this website and enter the URL of the problematic franchise website.
  • Check the firewall settings on the network, if the problem occurs on a LAN.
  • Determine if the network includes web filtering software.
  • Check the Windows HOSTS file to ensure no Blocked list is defined.
  • Attempt to access the site using the IP address.
  • Clear the computer’s DNS cache.
  • Inspect the site on the hosting server.

Avoiding Accidental Overwrites in GitHub

To avoid deploying outdated or incorrect code to production, it is critical that the Rollins Marketing Technology team and all Rollins contributors adhere to best practices when committing code to GitHub repositories.
See Mitigating Risk in GitHub Version Control for procedures and best practices.

IMPORTANT: Always Pull, then Push.
To ensure you are always working on the latest code base, pull from the dev.orkin.com repository, resolve any conflicts, then push the code back to dev.orkin.com.

Basic Troubleshooting Procedures

Many minor problems can be easily investigated or corrected using these basic procedures.


Pinging a Website

To ping a website, follow these steps:

  1. Open the Command Prompt in Windows. Click the Start button and enter cmd into the Search field. Press Enter to launch the Command Prompt.
  2. Type ping hostname - for example, type ping orkin.com
  3. A successful Ping command will return lines that display how long it took the address to respond.

Flushing the DNS Cache

To ensure rapid page loading, your computer automatically caches information about websites you visit. If a web page fails to load, the problem could be due to a DNS cache that is not updated. Flushing your DNS cache forces your computer to request current information from the website. To flush your DNS cache, follow these steps:
IMPORTANT: Only members of the Marketing Technology team should flush the DNS cache. These procedures may require administrator access to your computer.

Windows

  1. On your Windows computer, select Start and type cmd in the search field.
  2. Press Enter. A command prompt is displayed.
  3. Type ipconfig /flushdns and press Enter. A message confirms that the flush is successful.

Mac

  1. On your Mac computer, navigate to Applications > Utilities and double-click Terminal.
  2. Type - dscacheutil –flushcache and press Enter.
  3. Type sudo killall -HUP mDNSResponder and press Enter. This command reloads the DNS cache.

 

Inspecting the HOSTS file for Blocked Sites

The Windows HOSTS file can be used to block websites. If a Rollins employee or a franchisee cannot access a website, open the HOSTS file to ensure that website is not entered as a blocked site. To check the file, follow these steps:


IMPORTANT: Only members of the Marketing Technology team who are experienced in Windows administration should inspect this system file.

 

  1. Navigate to C:\Windows\system32\drivers\etc\ and locate hosts.
  2. Right click the hosts file and select Copy. This creates a backup copy of the file.
  3. Open hosts in a text editor and scroll to the end of the file.
  4. Below the code 127.0.0.1 local host, a list of blocked sites may be listed. If no sites are listed below localhost, the HOSTS file is not the cause of the issue. Close the file without saving.
    If blocked sites are listed below localhost, and the franchise website appears in the list, delete that line, and close and save the file.
