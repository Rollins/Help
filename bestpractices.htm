Cloud Architecture

Controllable

  • Decompose into small, loosely coupled, stateless building blocks
  • Automate your Application and Processes
  • Let Business levers control the system (the architecture should scale based on this architecture)
  • Architect with Cost in mind
  • Use Data sharding 

Resilient

  • Protecting your customer is the first priority (using encryption)
  • In production, deploy to at least two availability zones
  • Integrate security into your application from the ground up 
  • Build, test, integrate and deploy continously
  • Phase in deployments with multiple ec2 instances

Adaptive

  • Build the components with as less complexity as possible
  • Use late binding to decide on what instance type to use, rather than assuming at first, based on the memory and storage requirements

Data driven

  • Instrument everything all the time
  • Implement Logging, Monitoring of the applications as effectively as possible
  • Use benchmarking

Note: Building Stateless blocks makes autoscaling easy

Flexibility, Scalability and Measurability
A baseline reserved instance, performing scalability by using spot instances if they are not available go for on demand instances, all this part of the code needs to be automated

Information Architecture

Information Architecture standards are designed to ensure that Rollins web properties and mobile applications can make the most use, and reuse, of the gigabytes of assets available. The result of good IA is that assets are optimized for search engines to make a web page, image, video, or other resource on easy to find for anyone who's looking both internally and externally to the organization. Though search engines are the primary motivation for the IA of websites, a well designed structure will benefit the content creators, coders, and designers working on the website as well.

File Naming Rules

Good IA starts with good file names. Good file names are both descriptive and succinct. File names should...
  • Be descriptive of the file’s contents.
  • Be all lower-case (the CDN is case sensitive).
  • Not use acronyms, abbreviations, and shorthand spellings.
  • Be worded naturally, in “headline” style (avoiding words like “the”, “a”, and “and” when possible).
  • Avoid pronouns.
  • Use hyphens instead of spaces.
  • Not specify the type of file in any way other than the file extension (e.g. do not name a video bed-bugs-crawling-on-mattress-video.mp4, simply call it bed-bugs-crawling-on-mattress.mp4).
  • Use only alphanumeric characters, except where allowed by these rules.

Naming Image Files

File names for images should follow the same naming rules as other files, appended with color (optional) and dimension information as follows:

  1. Color (optional): The color of the image is optional and should be used primarily to distinguish between otherwise identical images.
  2. Dimensions: The dimensions should be listed in WxH format (width by height). For example: 10x20.

Each part of the image file name is separated by the underscore ("_") character. A hyphen ("-") is to be used in place of a space.

Examples of Image Names

  • ant-under-magnifying-glass_78x75.gif
  • tick_72x60.gif
  • navigation-item_grey_130x30.gif
  • navigation-item_red_130x30.gif

Directory Structure

The /images/ Directory

For any content published online, the directory for images should be "/images" (directly off the root of the domain). A sub-folder, /images/misc/, can be used for images that should not be crawled by bots, such as the Google Bot. 

Images of pests receive special treatment in the /images/ directory. Each pest image is to be placed in a sub-directory of the /images/ directory which indicates the family, or category, of pest the pictured pest belongs to. For example, a picture of house fly would be placed under /images/flies/ (an example of the full file name would be /images/flies/house-fly_50x50.jpg).

Content Quality

All content should exemplify high Rollins standards for quality and consistency; web content, in addition to adhering to brand standards, should adhere to SEO best practices. For text this means using descriptive words and phrases, appropriate headers, descriptive link text, and semantic code from schema.org when relevant. For images, this means finding the right balance between fast-loading and high-quality; in Photoshop this usually means using the "High" to "Very High" quality settings when saving out a jpg image.
