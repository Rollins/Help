Sidebars

A WordPress sidebar is a dynamic GUI for adding and formatting sidebar content.  The side bars are declared in the Orkin2011 theme file named functions.php inside of the following function:

function Orkin2011_widgets_init() {

                        register_sidebar( array(
                                'name' => __( 'Sidebar - Home', 'Orkin2011' ),
                                'id' => 'sidebar_home',
                                'description' => __( 'The Home Page Sidebar ', 'Orkin2011' ),
                                'before_widget' => '',
                                'after_widget' =>'',
                                'before_title' => '<h3 class="widget-title">',
                                'after_title' => '</h3>',
                        ) );

                        register_sidebar( array(
                                'name' => __( 'Share Media Panel', 'Orkin2011' ),
                                'id' => 'share_media_panel',
                                'description' => __( 'Share Media Panel', 'Orkin2011' ),
                                'before_widget' => '',
                                'after_widget' => '',
                                'before_title' => '<h3 class="widget-title">',
                                'after_title' => '</h3>',
                        ) );

                        register_sidebar( array(
                                'name' => __( 'Sidebar - Inner ', 'Orkin2011' ),
                                'id' => 'sidebar_inner',
                                'description' => __( 'Sidebar Inner Site (Not Home)', 'Orkin2011' ),
                                'before_widget' => '',
                                'after_widget' => '',
                                'before_title' => '<h3 class="widget-title">',
                                'after_title' => '</h3>',
                        ) );
                        
                                

                }

                add_action( 'widgets_init', 'Orkin2011_widgets_init' );

Once you have added the code to the functions.php file it will show up under the Appearance->Widgets page inside the admin panel of the site. As you can see in the code there has been a sidebar declared called Sidebar Commercial Home and then in the image below you will see that the widgets menu has Sidebar Commercial Home. Creating and Updating Pages

You create a page in In almost the exact same manner as you would a Pest Library Item.  
To create a new page go to Pages->Add New:


 

Once that is done you will see the following:


The key areas to note here are the

  • Title textbox
  • Visual/HTML tabs to toggle between visual and html  (currently the use of <figcaption> and toggling between Visual and HTML will cause the Editor to add unwanted &nbsp;  so the use of the <figcaption> is NOT supported)
  • Page Attributes : this section will allow you to choose whether there is a parent page and a template.
  • Page Parent : will the New Page be a sub-page of some other page?
  • Choosing a Template:  You will need to decide if the New Page will require a template.  By looking at the templates example you will be able to get a better idea of whether or not you will need one and which one to use.  NOTE: if you are using a template with a Callout box, you will need to save the page, then refresh (f5) the page and you will see then the Callout Tab at the top of the page.

Widgets

This section provides basic information on how widgets are instantiated and configured.
You can start all new widget tasks with the sample widget code found here.  Detailed instructions are provided with this code.

**  Instructions  -
**  Copy the entire widget-sample-widget folder and   
**  rename it to widget-[your-widget-name].  Rename the
**  sample-widget.php file to[your-widget-name]-widget.php.
**  Now open the [your-widget-name]-widget.php in your
**  favorite editor.  Using a global find/replace you can
**  easily rename all functions and variables to
**  [your_widget_name] NOTE: you must use UNDERSCORES!!!
**  PHP functions cannot have dashes in the name.  
**  So find (sample) replace with [your_widget_name].
**  Configure the output function [your_widget_name]_widget()
**  and the widget admin control [your_widget_name]_contol().
 
**  Once you are done, save the folder into the /wp-content/plugins directory.  Now in the admin panel select plugins and then activate the plugin.  

**  You can now find your new widget in the appearance->widgets  

 

Dynamic Sidebar Widgets

Once you have created your text sidebars or added a new sidebar you will need to configure how and where the widget will display. Navigate to your admin control panel and select Appearence->Dynamic Widgets.



You will then see the Dynamic Widget Control:



Notice that all sidebars are listed that have widgets attached to them. Once you locate the widget that you want to configure for display you will want to click on the widget to access the configuration console. NOTE: Too many text widget with no description? You can determine the correct one by looking in the Appearance->Widgets and counting down to the correct one, see below. In this example I have added a Text widget to the Sidebar - Inner.



Now below you will see that the text widget I just added will display under the correct sidebar in the Dynamic Widgets manager.



Now I can simply select the widget and configure the proper display. The main area you will want to focus on will be the "Pages" section. In this example I only want this page to display on two pages only. First I select the Radio button No for "Show widget default on static pages". Now I have selected two pages I want the widgets to display on.


It is not likely that you will ever have to use the rest of the configuration options, but the same basic principal for all configuration options. Now all that is left is to save the configuration and your dynamic widget will display when the page is called.

 

 

Adding Articles

Rollins Marketing Business Owners review contributions and approve them for deployment to production. Follow these guidelines:

  • After developing and testing your contribution in a local environment, push the code to dev.orkin.com.

  • Use VersionOne Conversations to notify the Business Owner that the contribution is ready for review.

  • Work with the Business Owner on adjustments and revisions. If necessary, push updated code to dev.orkin.com for the Business Owner’s approval.

IMPORTANT:  Use VersionOne Conversations to track revisions and review cycles.

  • Ensure the Business Owner approves the final contribution in VersionOne.
