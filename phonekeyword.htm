Phone and Web Leads Tracking

Rollins Marketing uses a keyword-to-call tracking solution that provides data collection and business intelligence functions, used to optimize SEM programs to the keyword level. This solution uses dynamic phone number insertion to accurately link online and offline behavior data.

About Lead Tracking

Reliable tracking that distinguishes between online and offline lead sources is important to marketing initatives. The phone and web leads tracking system enables Marketing to track almost every lead, sale, and start for Orkin.

Campaign codes are associated with a specific phone numer that identifies the source of the lead. A status is assigned to the lead, indicating whether the lead became a sale or start.

The tracking, analysis, and optimization of leads is critical to Digital Marketing decisions.

Marketing Technology uses a process workflow to combine data from several sources. This workflow gathers records such as caller data, session data, campaign code, and revenue data, then matches and combines this data for analysis in the Marin platform.

The Marketing team uses the Marin platform to analyze this data for lead optimization.

 

Tracking Process

The following process describes the conversion of data. Click here to see a diagram of this process.

 

  • Call data - such as Calling Party, VDN, and call start and end time - is stored in CMS.

  • Case data is collected in Contact360 when customer service representatives log information from customer inquiries. The Case ID is necessary for this process.

  • The AvidTrak call tracking platform stores session data, such as keyword information, creative ID, and match type.

  • Each day, the call data and case data is sent to to AvidTrak, and the session data is appended to the call and case data.

  • This combined data is moved to the Rollins Integration Marketing Database. The AvidTrak API is used to match calls with associated session data.

  • The Call, Case, and Session data is moved to the Marin platform. Form-fill leads data, stored in the Marketing Database, is also moved to Marin, as well as revenue data from other ODS sources.

Backend Systems and Processes

This section describes the backend processes that deliver robust data to the Marin platform.

These processes are executed daily by ODS and the Marketing Technology team. They generate and use the artifacts to implement the call matching process.

 

ODS: Generate Calls File Export

This process generates the AvidTrak TO file.

The name of this file is similar to AvidTrak_20150612040026.txt; it contains the following fields from Contact360 and CMS:

 

Column Name Description
CALLING_PTY The ten digit caller ID phone number of the calling party.
FIRSTVDN The first VDN the call was routed through. This ultimately indicates the number that was dialed.
CALLID The call identifier assigned by Contact360. This is also used as the primary key in the Calls table
SEGSTART The date and time the call was placed.
SEGSTOP The date and time the call ended.
CASEID The Contact360 Case ID tied to the lead.

 

This process begins at 4:00 AM daily and takes less than one minute.

 

 

CallsImport

After the AvidTrack TO file is generated, the CallsImport process retrieves it from the network share G drive and imports the calls in the Rollins Integration MDS Calls and CallCases tables. The CallCases table is a master detail table for calls associated with multiple cases. During the import process, the call is matched to a marketing campaign by the VDN associated with the dialed phone number. CallsImport also processes any available Calls File Export since the last successful import.

This process begins at 4:05 AM daily and takes one or two minutes, depending on the call volume of the previous day.

 

CallsCdrExport

After the calls are available in the Calls and CallCases tables, the CallsCdrExport process generates the Call Detail Record report.

The name of this file is similar to call-detail-report2016-11-25-09-10-01z.cdr; it contains the following fields:

 

Column Name Description
CallOriginatingNumber The ten digit caller ID phone number of the calling party.
CallDialedNumber The ten digit number of the dialed line.
CallConnectUtc The date and time (in UTC) the call was connected.
CallTerminateUtc The date and time (in UTC) the call was terminated.

 

This file is uploaded to the AvidTrak FTP directory for import into the AvidTrak portal.

This process begins at 4:10 AM daily and takes five to ten minutes, depending on the number of imported calls since the last export.

 

 

CallsAppend

After AvidTrak completes the import of the call details, the CallsAppend process uses the AvidTrak API to download all calls matched to an AvidTrak call. The CallBidKeywordText, KeywordId, CreativeId, MatchType values, if available, are appended to the existing call data in the Calls table. The CallClickDataAppended field is also set, indicating the call has been appended and additional processing will not append data to the call.

This process begins at 11:20 am daily. While the append process typically takes less than 15 minutes, the AvidTrak window for this processing task is seven hours.

IMPORTANT: Stakeholders should plan for a seven-hour turnaround time for this process to complete.

 

 

ODS: Generate Conversion Files

This process retrieves conversion data for both web and phone leads, matches it to keyword data stored in the Rollins Integration MDB Calls table, and exports the Marin Conversion Files.

The name of this file is similar to combined-conv-phone_rollins_orkin_km271vm0l0_20161123.txt. It is sourced from ODS and the Rollins Integration Marketing Database, and contains the following fields:

 

Column Name Description
Date The date by which all the other fields are aggregated (grouped by).
Keyword ID The Marin keyword ID.
Creative ID A value generated by the ad platform that uniquely identifies an ad unit.
Keyword The bid-upon keyword.
Match Type The match type of the bid that was triggered by the search query.
websold Conv The number of sales made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from web leads.
webstart Conv The number of starts made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from web leads.
webstart Rev The revenue, as indicated by the contract value, generated on the specified date for the specified keyword ID, creative ID, keyword, and match type that was generated from web leads.
phonelead Conv The number of cases associated with the call. Multiple leads for the same call.
phonesold Conv The number of sales made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from phone leads.
phonestart Conv The number of starts made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from phone leads.
phonestart Rev The revenue, as indicated by the contract value, generated on the specified date for the specified keyword ID, creative ID, keyword, and match type that was generated from phone leads.

 

This process begins daily at 12:00 AM and takes less than a minute.

 

 

OdsLeadDataFtpUploadToMarin

After the combined web and phone conversion file is generated, the OdsLeadDataFtpUploadToMarin process transfers the file to the Marin FTP directory. Marin then uploads the data and bid optimizations can begin.

This process begins daily at 12:10 AM and takes less than a minute.

Troubleshooting

If the tracking system fails to execute properly, use the following information to troubleshoot and correct any problems:

 

Dependencies

If the tracking system fails to match data, consider the following:

  • System clocks in the Call Management System and Contact360 must be synced to a reasonable accuracy.
  • The customer service agent must close the case within five minutes of the call end.
  • The case originating number must match the call originating number.

 

Risks

The Marketing Technology team relies on experience and expertise to mitigate risk and troubleshoot issues. If the tracking system fails at any point in the process, the team can investigate the following issues:

Lack of Network Connectivity - if network connectivity fails, all processes in the tracking system fail and no data is uploaded to Marin. Common causes include:

  • Corporate network is down
  • Access to firewall port blocked by IT
  • Security updates applied to a critical resource machine

 

ODSLeadUploadToMarin file fails to generate - typically, this is caused by AvidTrak failing to generate its file. Common causes include:

  • An interruption in the FTP service, such as an upgrade
  • Critical resource is unavailable, due to planned maintenance or system crash

 

Password change - a password change at any point in the system can cause the process to fail if it is not propagated to the appropriate domains and services.

Host system shutdown - if a host system crashes, is manually shut down, or shut down remotely by IT, the tracking system process fails.

AvidTrak API returns invalid XML format - if the AvidTrak API creates an improperly formatted XML, no data is uploaded to Marin. The Marketing Technology team should contact AvidTrak for assistance.

AvidTrak WebService slow or unresponsive - if the AvidTrak service is not responsive, the Marketing Technology team should contact AvidTrak for assistance.

 

Special Cases

The following special cases may require attention:

 

Multiple calls from the same originating number

Multiple calls from different customers with the same originating number typically indicates the originating number is from a call tracking platform. The actual originating number is unobtainable, as are numbers originating from a private, blocked, or unlisted phone number.

 

Multiple cases linked to a single call

Multiple services are sometimes sold during a call, resulting in multiple cases linked to a single number. In this scenario, the cases are listed separately and therefore the call is listed multiple times. Each listed case is counted, as appropriate, in the sold, start, and revenue data.

 

Multiple calls linked to a single case
