Call Conversion Tracking

Rollins Marketing uses a keyword-to-call tracking solution that provides data collection and business intelligence functions, used to optimize SEM programs to the keyword level. This solution uses dynamic phone number insertion to accurately link online and offline behavior data.

About Call Conversion Tracking

Reliable tracking that distinguishes between online and offline lead sources is important to marketing initatives. The call conversion tracking system enables Marketing to track almost every lead, sale and start for Orkin.

Campaign codes are associated with a specific phone numer that identifies the source of the lead. A status is assigned to the lead, indicating whether the lead became a sale or start.

The tracking, analysis, and optimization of leads is critical to Digital Marketing decisions.

 

 

Rollins Call Conversion Workflow

Marketing Technology uses a conversion workflow to combine data from several sources. This workflow gathers records such as caller data, click data, campaign code, and revenue data, then matches and combines this data for viewing in the Marin platform.

The Marketing team uses the Marin platform to analyze this data for lead optimization.

 

Call Conversion Overview

The following process describes the conversion of data. Click here to see a diagram of this process.

 

Import AvidTrack calls into RI-MDB - The daily process begins by importing calls into the Rollins Integration Marketing Database (RI-MDB). To determine which calls are associated with AvidTrak, each number in the AvidTrak replacement bank is associated to a unique Marketing Campaign and VDN. Each Marketing Campaign is associated with a Marketing Program. Marketing Programs have a field indicating its inclusion in AvidTrak.  

Export calls to AvidTrak - After calls are imported into the Rollins Integration MDB, they are then exported to AvidTrak and marked as exported.

Match exported calls in AT session - AvidTrak matches the start and end times of the recorded sessions to the call connect and terminate times of the exported calls.  

Match click data in RI-MDB - Click data is downloaded from AvidTrak paid client APIs and matched to calls stored in the Rollins Integration MDB. It is important to match only paid clicks, because only these clicks need to be exported to Marin. These calls now have their associated click data stored in the RI-MDB.

ODS generate conversion file - Using the VDNs associated with leads, ODS matches lead/sold/start data to call VDNs in the Rollins Integration MDB and generates a conversion file with revenue data. Matching is done on calls for 10 days previous to ensure enough time is given to close the lead/sold/start sales loop.

Process Artifacts

 

The CallsImport file is a TXT file sourced from CMS and Contact360. It is generated daily and imported to the Rollins Integration Marketing Database. The file contains the following fields:

Column Name Description
CALLING_PTY The ten digit caller ID phone number of the calling party.
FIRSTVDN The first VDN the call was routed through. This ultimately indicates the number that was dialed.
CALLID The call identifier assigned by Edge. This is also used as the primary key in the Calls table
SEGSTART The date and time the call was placed.
SEGSTOP The date and time the call ended.
CASEID The Edge Case ID tied to the lead.

 

The CallsExport file is a CDR file sourced from the Rollins Integration Marketing Database. It is imported to AvidTrack. This file contains the following fields:

Column Name Description
CallOriginatingNumber The ten digit caller ID phone number of the calling party.
CallDialedNumber The ten digit number of the dialed line.
CallConnectUtc The date and time (in UTC) the call was connected.
CallTerminateUtc The date and time (in UTC) the call was terminated.
 

The ODSLeadUploadToMarin file is a TXT file sourced from ODS and the Rollins Integration Marketing Database and imported to Marin. The web and phone conversion files contain the following fields:

 

Column Name Description
Date The date by which all the other fields are aggregated (grouped by).
Keyword ID The Marin keyword ID.
Creative ID A value generated by the ad platform that uniquely identifies an ad unit.
Keyword The bid-upon keyword.
Match Type The match type of the bid that was triggered by the search query.
websold Conv The number of sales made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from web leads.
webstart Conv The number of starts made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from web leads.
webstart Rev The revenue, as indicated by the contract value, generated on the specified date for the specified keyword ID, creative ID, keyword, and match type that was generated from web leads.
phonelead Conv The number of cases associated with the call. Multiple leads for the same call. Please see Linking a Call to a Case ID => Special and Unusual Cases below.
phonesold Conv The number of sales made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from phone leads.
phonestart Conv The number of starts made on the specified date for the specified keyword ID, creative ID, keyword, and match type that were generated from phone leads.
phonestart Rev The revenue, as indicated by the contract value, generated on the specified date for the specified keyword ID, creative ID, keyword, and match type that was generated from phone leads.

 

 

 

 

System Processes

These processes are executed daily by ODS and Marketing Technology. They generate and use the artifacts to implement the call matching process.

ODS: Generate Calls File Export

This process generates the CallsImport file described above.

CallsImport

After the CallsImport file is generated, the CallsImport process retrieves it from the network share and imports the calls in the Rollins Integration MDS Calls and CallCases tables. The CallCases table is a master detail table for calls associated with multiple cases. During the import process, the call is matched to a marketing campaign by the VDN associated with the dialed phone number. CallsImport also processes any available Calls File Export since the last successful import.

CallsCdrExport

After the calls are available in the Calls and CallCases tables, the CallsCdrExport process generates the Call Detail Record Export. This file is uploaded to the AvidTrak FTP directory for import into the AvidTrak portal.

CallsAppend

After AvidTrak completes the import of the call details, the CallsAppend process uses the AvidTrak API to download all calls matched to an AvidTrak call. The CallBidKeywordText, KeywordId, CreativeId, MatchType values, if available, are appended to the existing call data in the Calls table. The CallClickDataAppended field is also set, indicating the call has been appended and additional processing will not append data to the call.

ODS: Generate Conversion Files

This process retrieves conversion data for both web and phone leads, matches it to keyword data stored in the Rollins Integration MDB Calls table, and exports the Marin Conversion Files.

OdsLeadDataFtpUploadToMarin

After the combined web and phone conversion file is generated, the OdsLeadDataFtpUploadToMarin process transfers the file to the Marin FTP /km271vm0l0/combined-conv-phone directory. Marin then uploads the data and bid optimizations can begin.

Linking a Call to a Case ID

Linking a Call to a Case ID represents the key challenge in generating the ODS Calls File Export. Though Edge records the originating number of an in-bound call on the case it creates, it does not distinguish the specific call that generated the case from other calls from the same originating number. Therefore, to tie an Edge case to a a call in CMS the timestamp of the call and the timestamp of the Edge case must be "ligned up" to correlate the two.

Risks and Challenges

When a call comes in to the RCSC, an Edge case is automatically initiated but not created until the agent explicitly saves the record. Therefore the Edge case may be created any time during or after the call, preempting a simple correlation. Agent's spend on average two minutes on "after call work" and may not save the case until the after call work is completed.

Finally, Edge and CMS are on the same Time Zone but should not be assumed to be synchronized with one another, or even to have a consistent difference (a reboot may cause one system to become more or less synchronized with the other).

Assumptions

  • The clocks between Edge and CMS are accurate within plus-or-minus two minutes of each other.
  • An agent will spend no more than 5 minutes on after call work.

Matching Cases to Calls

A call can be linked to a case if:

  • The Edge case's originating number matches the call record's originating number.
  • AND the Edge case creation timestamp is greater than or equal to two minutes prior to the call connection time.
  • AND the Edge case creation timestamp is less than or equal to seven minutes after the call termination time.

Which Calls to Consider

Only calls to active phone numbers or VDNs in the Digital Acquisition marketing program should be considered for tying call records to Edge cases, all other data can be discarded for the purposes of Liminal. It may later be necessary to include all active phone numbers and VDNs, should the reporting structure ever be split out. A phone number or VDN is considered "active" in the RollinsIntegration.MarketingCampaigns table if the MarketingCampaignStartDateTime value is less than or equal to the current time AND the MarketingCampaignEndDateTime is greater than the current time OR null.

Special and Unusual Cases

Same Phone Number

This scenario occurs when a multiple calls from different customers are generated by the same originating number.

Example Data

Case # Type Created Originating Number Call Connected Call Terminated Location
21238122 LEAD 2013-06-01 08:49 303-444-7428 2013-06-01 08:39 2013-06-01 08:49 Cincinnati, OH 45241
21238986 SALE 2013-06-01 11:10 303-444-7428 2013-06-01 10:56 2013-06-01 11:10 Peoria, AZ 70203

Note that the calls from 303-444-7428 came in from two different locations for two different on the same day.

Explanation

The originating number is from a call tracking platform, Telmetrics in this case, and the actual originating number was unobtainable. Originating numbers may be unobtainable if the calling party is dialing from a private or unlisted phone number. (FW  303-444-7428.msg)

How to Handle

These calls require no special handling.

Mutliple Cases for a Single Call

This scenario occurs when multiple services are sold during a single service call. Therefor the call is matched to multiple cases in Edge.

Example Data

Case # Type Originating Number Call Connected Call Terminated Customer
21238517 SALE 555-555-5555 2013-06-01 09:44 2013-06-01 10:11 Jane Doe
21238642 LEAD 555-555-5555 2016-06-01 09:44 2013-06-01 10:11 Jane Doe

Note that the same call generated both cases.

How to Handle

The ODS Calls File Export must list both cases; in other words, it should list the same call twice. The Marin Conversion File Export must count both cases appropriatly in the sold, start, and revenue numbers. For example, if both cases become sales but only one becomes a start then the "phonesold Conv" and "phonestart Conv" values would be incremented by 2 and 1, respectively.

Multiple Calls for a Single Case

This can occur when a customer calls back and references a service/lead created earlier.

How to Handle

The ODS calls file export will list the same call twice, however each will have its own case ID. When the Marin conversion file is created, however, the call will only be listed once, with the phonelead Conv column incremented for each call.

Linking Clicks to Calls

The ODS Calls File Export will link calls to Edge cases. This data is imported into RollinsIntegration then exported to AvidTrak via the Call Detail Record (CDR) Export. AvidTrak processes the CRD file by matching the records in the file against its phone line pool allocation history. For this project, it is not important how AvidTrak works, only that it works. Once AvidTrak has performed the matching, the appended data is loaded back into RollinsIntegration.

AvidTrak requires roughly 30 minutes to match call data to click data and incorporate the details into reports. To be safe, click data should not be loaded until at least 60 minutes after it has been loaded into AvidTrak.

Linking Click-and-Call Data to Sales, Starts, and Revenue

Once click data has been downloaded from AvidTrak and appended to the call data that was imported from ODS, thus forming combined records of click and call data, which will include the Edge case ID. ODS then loads the appended data, using the Edge case ID to determine the sale and start count, as well as contract value (revenue amount).

Risks

Network Connectivity - if for some reason the network isn't available at any point during the process, the whole process will fail and no data can be uploaded to Marin. This occurred under the following situations:

  • When we switched office buildings in November 2013.
  • In December 2013, IT revoked access to port 1433 in the firewall, blocking us from accessing the necessary Azure SQL Server database.
  • During routine security updates to Daniel's computer, currently hosting the process.

ODS Call File Export file not generated - Occasionally the daily AvidTrak file isn't generated properly. In this situation the process will fail and no data will be uploaded to Marin. This occurred in the following situations:

  • 12/20/2013 - when IT upgraded the FTP service. In the future this should be coordinated with IT.
  • 12/4/2013 - Per Michele: "Focus is working on data updates that will run thru the night tonight, so we cannot run ODS. You’ll be able to run reports tomorrow, but they will not have today’s data. We should be able to catch-up tomorrow night for reporting on Friday."

Password Change - Domain requirements specify regular password resets. If the scheduled tasks are not updated with the new password the process will fail and no data will be uploaded to Marin.

Host system shut-down - 

  • manual shut down
  • remote shut down from IT
  • system crash

AvidTrak API returns improperly formatted XML - This has occurred twice:

  • first during the initial testing phase, and second in production on 1/3/2014.
  • Both times were quickly resolved by Amin and his team at AvidTrak and successive API calls were successful.

Liminal Process Import Exception and Trouble shooting and fixing the import Issue - the process fails to update the last successful run date that occurs due to some exception occurs at the run time.

Request Timeout - this exception occurs due to the AvidTrak Web Service fail to response in timely manner. it may happen due to the AvidTrak Server was not responding to the calls made by the web services. this error occurred on 12/31/2014

Linking Calls to Case IDs

The AvidTrack API provides:

  1. The caller's phone number (via Caller ID: caller).
  2. The number called (the solution's provided number: tracking_number).
  3. Timestamps for when the call was dialed (call_datetime) and ended (duration).
  4. Markeiting Campaign ID
  5. Mkwid
  6. Creative ID (adid).
  7. Bid Keyword Text (search_keyword).
  8. Match Type (match_type).

A Call Detail Record (CDR) feed from RCSC must contain the following information:

  1. The caller's phone number (via Caller ID).
  2. The number of the line dialed.
  3. Time-stamps for when the call was received and answered. Preferably in UTC.
  4. The Case ID associated with the call.

The CDR feed is loaded to AvidTrak, matching each call to keyword, creative, and other tracked fields. This data is loaded from AvidTrak to RollinsIntegration, and ultimately back into ODS where it can be linked to case status (lead, sold, start) and the revenue file can contain phone data.
